The repeating 2-hour day

Mon–Fri · build days

warm up → build → read
0:00–0:25 Foundations warm-up — one course lesson early on, then one NeetCode problem in the week's pattern.
0:25–1:45 Build — the current CodeCrafters stage. The anchor; most of the hour goes here.
1:45–2:00 Read — a few pages of the week's anchor chapter; jot one note toward the log.

Saturday · ship day

refactor → read → log
0:00–1:00 Close the week's build — finish or refactor the current stage; clear anything stuck.
1:00–1:30 Reading catch-up — the "light" book (APoSD, then OSTEP / Database Internals).
1:30–2:00 Write the decision log — the real artifact. This is what turns three tracks into one skill.

How to read this schedule

  • It's loose on dates, strict on order. Build hours are CodeCrafters' own estimates — if a stage takes longer, slip the dates and keep the sequence. The order is what makes each chapter and pattern land where it explains the build.
  • Build is the spine; everything else hangs off it. On any given day, most of the 2 hours is the build. Reading and NeetCode are paired to whatever milestone they illuminate, not run as separate marathons.
  • Saturday ships the artifact. The decision log is the deliverable — three tracks rolling up into one defensible call. Miss a build stage and recover; don't skip the log.
  • Database Internals & OSTEP can trail the build. They're deeper than the build strictly needs; DDIA and APoSD finish within the 24 weeks, the rest carries past it. Advanced Algorithms is explicitly optional.
  • Foundations is front-loaded. The three courses cluster in the ramp + M1 so that by SQLite onward, more of each day's 2 hours can go to the build.
